BINGO Game — Terminal-Based C Programming Project

A fully functional, feature-rich terminal BINGO game built entirely in C, running on Ubuntu Linux. It supports two game modes, background music, cheer sounds, a full-color terminal UI, and smooth animations without any game engine or external framework.

Game Modes

  • Player vs Player — Both players enter their chosen number directly in the terminal on each turn, enabling head-to-head play on the same machine.
  • Player vs Computer — The player enters a number each turn, while the computer selects randomly from the remaining available numbers to create a competitive opponent.

Key Features

  1. Background Music via Threads — Music plays continuously in parallel with gameplay using POSIX threads, so audio never blocks the main game flow.
  2. Cheer Sound on Line Completion — A short cheer clip plays whenever a player completes a row, column, or diagonal and earns a BINGO letter.
  3. Colorful Terminal UI — ANSI escape codes are used to color the grid, highlight marked numbers, and display earned BINGO letters in distinct colors.
  4. File Handling Animations — Intro screens, turn transitions, and win celebrations are driven by sequentially reading stored animation frames from files.
  5. Complete Grid Logic — All five rows, five columns, and two diagonals are checked after every move, with BINGO letters awarded in order from B to O.
  6. Computer AI — The computer picks from the pool of remaining unmarked numbers using randomized logic to avoid predictable behavior.

Technical Implementation

Component Implementation
Language C (GCC on Ubuntu)
Environment Ubuntu Linux Terminal
Threading POSIX Threads (pthreads) for background music
Colors ANSI escape codes for full-color terminal UI
Animations File handling with sequential frame reads
Grid Logic 2D arrays with row, column, and diagonal checks
Computer Logic Randomized selection from the remaining available number pool
Input Terminal stdin with turn-based flow control

Challenges & Solutions

  • Audio running in parallel without freezing — Solved by using pthreads so music runs independently from gameplay.
  • Animations in a plain terminal without graphics — Solved by reading and printing animation frames from files with controlled delays.
  • Making the terminal UI colorful and readable — Solved using ANSI escape codes for the grid, numbers, and BINGO letters.
  • Making the computer opponent feel natural — Solved with randomized selection from remaining available numbers.

What I Learned

  • Practical use of POSIX threads for parallel execution in C.
  • Creative use of file handling in C for animation beyond basic read and write operations.
  • Using ANSI terminal escape codes to build a rich UI without a GUI framework.
  • How to structure a multi-module C project across game logic, input, audio, and display.
  • How to design a user experience around timing, feedback, and presentation rather than functionality alone.

Development Environment

  • OS: Ubuntu Linux
  • Compiler: GCC
  • Threads: POSIX pthreads (-lpthread)
  • Audio: Linux audio system via terminal calls
  • Editor: Terminal / VS Code
