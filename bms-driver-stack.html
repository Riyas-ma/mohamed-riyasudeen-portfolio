Production BMS Driver Stack

Overview

Production-level BMS firmware developed for the Royal Enfield Flying Flea C6 electric motorcycle's in-house Battery Management System. Built to MISRA C and ASIL-D standards on the NXP S32K322 microcontroller with FreeRTOS, managing a 28S1P pack (28 cells in series, each cell being 1S8P — 8 parallel cells). The firmware covers complete cell monitoring, pack monitoring, passive balancing, low-power cell monitoring mode (LPCM), diagnostics, functional safety, and fault management — all developed from scratch based only on ADI reference code intended for evaluation purposes only.

The in-house BMS has completed homologation and is targeted for production release by end of 2026, replacing the currently outsourced BMS in the Flying Flea C6.

System Architecture

Daisy Chain Communication Order:

NXP S32K322 (MCU)
↕ SPI
ADBMS6822 (IsoSPI Bridge IC)
↕ IsoSPI
ADBMS6832 #1 (AFE1)
↕ IsoSPI
ADBMS6832 #2 (AFE2)
↕ IsoSPI
ADBMS2950 (Pack Monitor IC)

The MCU communicates with ADBMS6822 over SPI. ADBMS6822 converts SPI to IsoSPI differential signals. The three downstream ICs (AFE1, AFE2, ADBMS2950) are daisy-chained over IsoSPI.

IC Roles & What Was Developed

ADBMS6832 — Cell Monitoring AFE (×2 ICs)

Each IC monitors 14 cells differentially (28 cells total across both ICs).

  • Complete CDD (Complex Device Driver) — from low-level SPI/IsoSPI register access up to application layer
  • Cell voltage acquisition and conversion from raw ADC values to accurate millivolt readings
  • Temperature monitoring via 5 NTC thermistors per IC (10 NTCs total) using GPIO auxiliary ADC channels
  • Passive cell balancing using internal MOSFETs — PWM balancing method used to keep bleeding resistor temperature within safe range (supports up to 300 mA per cell)
  • Open-wire detection for all cell inputs and GPIO channels using the IC's redundant ADCs.
  • Internal IC parameter monitoring — triggers Cell Monitor Failure Fault if any abnormality detected
  • Low Power Cell Monitoring Mode (LPCM) — complete CDD from scratch including:
    • Configurable heartbeat period — IC wakes periodically, monitors, and returns to sleep
    • OV/UV threshold configuration for cell voltages and GPIO channels
    • INTR pin assertion handling for fault wake-up signaling
    • Cell balancing support during LPCM with configurable timeout as failsafe
    • Full LPCM enable/disable sequence with all hardware and software pre-conditions satisfied

ADBMS2950 — Pack Monitor IC (×1 IC)

  • Battery pack current measurement via dual shunt input channels (I1 & I2) using 18-bit ADC — raw to ampere conversion
  • Battery pack voltage and load voltage measurement via VBAT1 and VBAT2 channels
  • Isolation resistance measurement — complex multi-step logic for readng data before calculation implemented
  • Shunt temperature monitoring for shunt temperature compensation to improve battery current sensing accuracy
  • Fuse voltage sensing to detect fuse failures (charge fuse & discharge fuse)
  • GPIO and GPO control for switching measurement input MOSFETs during low-power mode to reduce power consumption
  • Open-wire diagnostics for all ADC inputs (VBATx, Vx, Ix) — shunt sense short and open circuit detection implemented
  • HV line Over-current / Short Circuit detection — OCA/OCB outputs assertion based on the configured threshold to blow the pyro fuse or to open the contactors instantly
  • Internal IC parameter monitoring — triggers Pack Monitor Failure Fault if any abnormality detected
  • Power supply management — load switch controlled by MCU to cut IC power during BMS low-power mode (IC has no native low-power mode)

Low-Power Mode & LPCM

This was the most complex and time-consuming part of the entire project.

What Was Implemented

  • ADBMS6832 LPCM enabled with full pre-condition satisfaction (both hardware and software conditions)
  • PMIC set to LPOFF mode after confirmed LPCM entry — all LDOs off, MCU powered down
  • PMIC LDT (Long Duration Timer) configured for periodic MCU wake-up as primary wake mechanism:
    • Every 5 minutes (Configurable)     —   when cell balancing is active (delta voltage above threshold)
    • Every 30 minutes (Configurable)  —   when balancing is inactive
  • Cell balancing timeout set to 6 minutes (Configurable) in ADBMS6832 as failsafe — if 5-minute wake-up does not occur, balancing auto-stops at 6th minute preventing over-discharge of cells

The PMIC Wake-Up Problem — Hardware Debug

A critical hardware issue was discovered and resolved during LPCM bring-up:

  • The ADBMS6832 INTR pin was connected in parallel to both the PMIC WAKE1 pin and the MCU GPIO (PTC12)
  • When a fault triggered in LPCM, INTR asserted — but the MCU (even in power-off state) drew leakage current through the GPIO pin
  • This pulled the INTR voltage down to approximately 600 mV — below the PMIC wake-up threshold
  • Result: PMIC could not wake up, MCU never turned on, faults went unhandled
  • Root cause identified by disconnecting MCU GPIO from INTR line — INTR voltage recovered to 3.4V, PMIC woke up correctly
  • Resolution: MCU GPIO connection to INTR removed. PMIC LDT periodic wake-up configured as the primary wake mechanism instead of fault-triggered wake
  • Hardware verified using oscilloscope — LDO1, PMIC, INTR, and IsoSPI signal lines captured and analyzed
